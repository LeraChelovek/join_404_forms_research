# Приложение для анализа пользовательского взаимодействия с веб-формами и страницами ошибки 404

Веб-приложение предназначено для проведения UX-эксперимента с применением трекинга взгляда, анализа взаимодействия пользователя с формами, проверки влияния прогресс-бара и оценки восприятия формальной и креативной страницы ошибки 404.

Приложение реализует сценарий виртуального кинотеатра: участник знакомится с инструкцией, проходит калибровку айтрекинга, выбирает фильм, сталкивается со страницей ошибки, заполняет экспериментальные формы и проходит итоговый опрос.

## Системные требования

Для корректной работы приложения необходимы:

- браузер Google Chrome или Mozilla Firefox;
- разрешение на использование веб-камеры;
- Python версии 3.10 или выше для запуска локального сервера и аналитических скриптов;
- стабильное освещение рабочего места;
- веб-камера, встроенная или внешняя.

## Установка и запуск

Клонируйте репозиторий из ветки final:

git clone <URL-репозитория>

Перейдите в папку проекта:

cd <имя-папки-проекта>
Установите Python-зависимости

Откройте приложение в браузере:

http://localhost:8000/cinema_index.html

Рекомендуется запускать приложение через локальный сервер, а не открывать HTML-файл напрямую, так как в проекте используется динамическая загрузка HTML-шаблонов форм.

## Калибровка айтрекинга
Перед началом основного сценария необходимо пройти калибровку системы отслеживания взгляда.

Для этого:

разрешите браузеру доступ к веб-камере;
нажмите кнопку начала калибровки;
поочередно нажимайте на появляющиеся контрольные точки;
старайтесь не двигать головой во время калибровки;
дождитесь завершения настройки WebGazer.
Калибровка необходима для повышения точности определения координат взгляда пользователя на экране.

Собираемые данные
В процессе эксперимента автоматически сохраняются данные айтрекинга.

## Рекомендации по проведению эксперимента

Используйте браузер Chrome или Edge.
Перед началом убедитесь, что веб-камера работает корректно.
Обеспечьте хорошее освещение лица участника.
Во время калибровки не двигайте головой.
Не обновляйте и не закрывайте страницу во время эксперимента.
Для каждого нового участника используйте новый уникальный ID.
После завершения эксперимента сохраните выгруженные CSV-файлы.
Назначение проекта
Проект может использоваться для исследования пользовательского опыта, оценки удобства веб-форм, анализа визуального внимания и изучения восприятия различных вариантов страниц ошибки 404.

## Основной исследовательский интерес связан с проверкой следующих гипотез:

Прогресс-бар повышает субъективное чувство контроля пользователя при заполнении формы.
Креативные страницы ошибки 404 воспринимаются пользователями более положительно, чем формальные.
Креативное оформление ошибок может быть полезным для бизнес-интерфейсов, так как помогает сохранить вовлеченность пользователя после возникновения сбоя.
